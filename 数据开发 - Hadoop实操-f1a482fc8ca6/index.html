  • 1.3 集群时间同步

  • 内网环境下需要配置（需要root权限），选一台节点作为时间服务器，其他节点进行同步

  • 配置时间服务器

    1. 关闭ntpt服务

      1. 临时关闭服务：systemctl stop ntpd
      2. 关闭开机自启动：systemctl disable ntpd

    2. 修改配置文件ntp.conf：vim /etc/ntp.conf

      1. # 修改：授权可同步的节点网段
restrict 192.168.10.0 mask 255.255.255.0 nomodify notrap

# 修改：注释互联网时间
#server 0.centos.pool.ntp.org iburst
#server 1.centos.pool.ntp.org iburst
#server 2.centos.pool.ntp.org iburst
#server 3.centos.pool.ntp.org iburst

# 添加：丢失网络时间后启用本地时间
server 127.127.1.0
fudge 127.127.1.0 stratum 10
    1. 开启ntpd服务
      1. 临时开启ntpd服务：systemctl start ntpd
      2. 开启开机自启动：systemctl enable ntpd

  • 配置其他服务器同步时间

    1. 关闭ntpt服务
      1. 临时关闭服务：systemctl stop ntpd
      2. 关闭开机自启动：systemctl disable ntpd
    2. 配置定时任务以同步时间：crontab -e
  • # 例：每分钟同步
*/1 * * * * /usr/sbin/ntpdate hadoop102

三、Hadoop操作

  1. 启动/停止集群

  2. 首次启动

    1. 格式化NameNode（NameNode节点执行）：hdfs namenode -format
    2. 同步NameNode元数据信息（其他NameNode节点执行）：hdfs namenode -bootstrapStandby

格式化会产生新的集群id，所以不能重复格式化，重复格式化需要先停止NameNode和DataNode进程，并删除所有节点data和logs目录

  1. 启动/停止集群

    1. 启动/停止HDFS：
      1. HDFS整体（NameNode节点执行）：start/stop-dfs.sh
      2. NameNode（NameNode节点执行）：hdfs --daemon start/stop namenode
      3. DataNode（DataNode节点执行）：hdfs --daemon start/stop datanode
      4. SecondaryNameNode（SecondaryNameNode节点执行）：hdfs --daemon start/stop secondarynamenode
      5. journalnode（journalnode节点执行）：hdfs --daemon start/stop journalnode
    2. 启动/停止YARN：
      1. YARN整体（ResourceManager节点执行）：start/stop-yarn.sh
      2. ResourceManager（ResourceManager节点执行）：yarn --daemon start/stop resourcemanager
      3. NodeManager（NodeManager节点执行）：yarn --daemon start/stop nodemanager
    3. 启动历史服务器（NameNode节点执行）：mapred --daemon start/stop historyserver
    4. 启动journalnode服务（journalnode节点执行）：hdfs --daemon start/stop journalnode

  2. 查看集群状态：jps

  3. 操作集群

  1. 本地运行：配置环境后直接运行程序

    1. 编写程序，打jar包

  3. 本地提交到集群运行

    1. Driver类添加必要配置信息

    2. Configuration conf = new Configuration();
//设置在集群运行的相关参数-设置HDFS,NAMENODE的地址
conf.set("fs.defaultFS", "hdfs://hadoop102:8020");
//指定MR运行在Yarn上
conf.set("mapreduce.framework.name","yarn");
//指定MR可以在远程集群运行
conf.set("mapreduce.app-submission.cross-platform","true");
//指定yarn resourcemanager的位置
conf.set("yarn.resourcemanager.hostname","hadoop103")

    3. 程序编写完毕后，打jar包

    4. 将jar包设置到Driver，参数为jar包绝对路径

    5. job.setJar("C:\\Users\\skiin\\IdeaProjects\\mapreduce1021\\target\\mapreduce1021-1.0-SNAPSHOT.jar");

    6. source /etc/profile.d/my_env.sh
